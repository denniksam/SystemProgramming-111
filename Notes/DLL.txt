DLL - Dynamic Link Library

Библиотеки кода - отдельные файлы с кодами дополнительных функций/классов
Делятся на
 1 - библиотеки статической компоновки (как в С++) - содержимое файлов
    добавляется как текст к нашей программе, после чего компилируется 
    собранный файл;
 2 - библиотеки динамической компоновки - скомпилированные файлы, в которых
    код готов к исполнению, другие программы могут использовать этот код
    для запуска в составе своих приложений.


1 :
 "+" файл программы один, все необходимое находится в нем
 "-" в файл добавляются все функции из библиотеки, независимо от того
      используются они или нет
 "+" меньшая зависимость от ОS, для адаптации достаточно перекомплировать
      программу
2 :
 "-" для запуска программы необходимо наличие всех используемых DLL
 "+" библиотека одна, все файлы могут к ней обращаться, ее коды никак
      не изменяют размер программ
 "-" исполнимость сильно зависит от OS, поэтому DLL практически невозможно
      переносить между ними
-----
в виде DLL реализуют системные функции - они поставляются с OS,
регистрируются в папках общего доступа, скомпилированы в полной адаптации
с OS, сохраняя названия и имена функций для использования в других программах.
Совокупность инструментов OS для прикладных программ называют API
(Application - Program Interface) - WinAPI


!! Расширение файлов .dll имеют как реально исполнимые файлы (те же .exe файлы,
только без main() - без точки входа),
так и библиотеки классов .NET, которые являются промежуточным (IL) кодом и
исполняются JIT платформы.

Marshal - Маршализация - процесс взаимодействия (обмена данными) между
управляемым и неуправляемым кодом, interoperability

                 Marshal
Unmanaged <-----------------------> Managed
EXE/DLL - OS                        IL - .NET/Class Lib
DWORD                               System.Int32
BYTE                                System.Byte
int*                                System.IntPtr
https://learn.microsoft.com/en-us/dotnet/framework/interop/marshalling-data-with-platform-invoke

Как использовать функцию из DLL
нужно знать название файла .dll, его размещение (если он не в общей папке)
            полное название функции и ее параметров,
            типы данных возврата и параметров, сопоставление этих типов с .NET

Например, MessageBoxA - системная функция для вывода сообщения
- библиотека (файл) User32.dll
- сигнатура
    int MessageBoxA(
      [in, optional] HWND   hWnd,
      [in, optional] LPCSTR lpText,
      [in, optional] LPCSTR lpCaption,
      [in]           UINT   uType
    );
- сопоставление типов
int       int (System.Int32)
HWND      System.IntPtr
LPCSTR    String
LPCSTR    String
UINT      uint
--------------------------------------------------
